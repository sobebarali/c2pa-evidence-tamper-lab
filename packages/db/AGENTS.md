# db (`@c2pa-evidence-tamper-lab/db`)

> Drizzle schema + queries over libSQL/SQLite (Turso), plus the in-memory test harness. Testing doctrine: [`../config/AGENTS.md`](../config/AGENTS.md).

## Surface

- `src/index.ts` — `createDb()` + the `db` singleton (env-driven) + the `Db` type. Handlers never import the singleton; it's injected via context.
- `src/schema/index.ts` — the tables.
- `src/testing.ts` — `createTestDb()`: in-memory libSQL + `pushSQLiteSchema` (the SQLite analog of fomo's PGlite). Returns `{ db, close }`.
- `src/migrations/` — generated by `bun run db:generate`.

## Tables

| Table | Role |
|-------|------|
| `files` | addressable index of stored image bytes (disk under `DATA_DIR`, served via `GET /files/:id`). `kind` ∈ original/signed/tampered/uploaded; captures sha256/mime/size/dims. |
| `evidence_records` | one row per signed evidence (task.md §5): `evidenceId` (pk), mode, original/signed file ids + hashes, manifest label, claim generator, signature status, `validationErrors` (json), `extractedEvidenceJson` (json). FKs → `files.id`. |

## Conventions (Rule → Why)

| Rule | Why |
|------|-----|
| DB is not cryptographic truth | the C2PA verifier decides validity; the DB only links an upload to a prior record by `evidenceId` (task.md §5) |
| json columns via `text({ mode: "json" }).$type<…>()` | typed read/write without a second serializer |
| timestamps `integer({ mode: "timestamp" })` default `unixepoch()` | DB-side default, read back as `Date` |
